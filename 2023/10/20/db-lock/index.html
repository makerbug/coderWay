MySQL锁

发表于 2023-10-20 分类于 db
详解MySQL各种类型的锁

1. 锁的分类

多个事务或进程访问同一个资源的时候，为了保证数据的一致性，就需要用到锁机制。MySQL的锁在服务器层或存储引擎层实现

1.1 锁定资源的不同

根据锁定资源的不同，MySQL中的锁分为：

资源就是数据，MySQL提供插件式引擎对数据进行存储（Innodb、MyISAM）

  • 表级锁
  • 行级锁
  • 页面锁
  • 全局锁:对整个数据库实例加锁

1.2 模式的不同

1.2.1 乐观锁
(1) 概念

乐观锁假设数据一般情况下不会造成冲突，所以数据进行提交更新的时候，才会正式对数据的冲突与否进行检测，如果发现冲突了，就返回给用户错误的信息，让用户决定如何去做。

(2) 应用场景

适用于读多写少的场景（大量的写操作增大写冲突的可能性,业务层需要不断重试，降低系统性能）

(3) 实现方式

版本号机制

版本号机制：在数据库表中增加一个数字类型的Version字段来实现。
每次读取数据时，将Version字段的值一同读出，数据每更新一次，Version的值就加一。
提交更新的时候，判断数据库表对应记录的Version与第一次取出Version值进行比较，若一致，则予以更新。不一致那必然是有人在中间修改过。

(4) 实战 一定要有实战!!

1.查询订单信息
select id,status,version from order where id=#{id};
2.用户支付成功
3.修改订单状态
update set status=支付成功,version=version+1 where id=#{id} and version=#{ version};

1.2.2悲观锁

(1) 概念

悲观锁是一种具有强烈独占和排他特性的生物，他小心翼翼，认为每次被外界(本系统当前的事务，外部系统的事务处理)读取数据的时候数据都会被修改，因此在整个数据处理过程中，数据处于锁定状态。

(2) 应用场景

并发量不大，写入操作频繁，数据一致性较高的场景

(3) 实现方式

使用悲观锁，必须关闭MySQL的自动提交

MySQL默认开启事务自动提交模式，除非显示的开启事务(BEGIN 或 START Transcation),否则每条SQL语句都会被当作一个单独的事务自动执行。 set autocommit=0

共享锁和排他锁是悲观锁的不同时实现

(4) 实战
  1. 开启事务并关闭自动提交
    start transaction;
    set autocommit =0;
  2. 查询商品信息
    select id,name,number from goods where id = 1 for update

    MySQL中的for update
    for update可以为数据库中的行上一个排它锁，当一个事务操作未完成的时候，其他事务可以读取但不能写入或更新。
    如果查询条件中带有主键，会锁行数据，如果没有，会锁表
    MySQL InnoDB默认行级锁，行级锁都是基于索引的，如果一条语句用不到索引是不会使用行级锁的。

  3. 用户下单，生成订单
  4. 修改商品库存
    update set number = number-1 where id =1
  5. 提交事务
    commit

1.3 属性的不同

1.3.1 共享锁

(1) 概念

共享锁：读锁、S锁
事务A对数据加了S锁，其他事务只能读不能写，直到A释放读锁

(2) 应用场景

并发读取数据
两个表存在关系时的写操作

(3) 实现方式

select * from goods where id =1 lock in share mode;

1.3.2 排它锁

同上面的for update

1.4 意向共享锁和意向排它锁

(1) 概念

意向锁是表锁，为了协调行锁和表锁的关系，支持多粒度（表锁与行锁的并存）

当事务A有行锁时，MySQL会自动为该表添加意向锁，事务B如果想申请整个表的写锁，那么不需要便利每一行是否存在行锁，而是直接判断是否存在意向锁，增强性能。

(2) 意向锁的兼容互斥性
意向共享锁(IS) 意向排它锁(IX)
共享锁(S) 兼容 互斥
排它锁(X) 互斥 互斥
(3) 实战 实战一定要有

Session1获取id=1数据行的锁但是没有提交

此时User表存在两把锁，User表上的意向排它锁和id为1数据行上的排它锁。
Session2想要获取User表的共享锁
LOCK TABLES User READ
此时Session2对User表的加锁请求就会被阻塞

1.5 间隙锁、记录锁、Next-Key locks(临键锁)

(1) 概念

都是排它锁

(2) 记录锁

select * from goods where id = 1 for update

(3) 间隙锁

select * from goods where id between 1 and 10 for update
所有在(1,10)区间内的记录行都会被锁住 1,10不会

(4) Next-Key Locks

记录锁+间隙锁 左开右闭的区间 防止幻读
每个数据行上的非唯一
